hexo-블로그엔비증

2020-03-17
블로그, hexo, 오류

이건 뭔가?

1. hexo 터미널서 이상하다.

정말 모를 일이다. 어제 했던게 모두 날라가버린 기분이다.
나는 정말 차분하게 터미널에 이러게 쳤다.

git remote -v

그런데 이렇게 나온다.

어처구니가 없다. 그래서 이것도 해본다. hexo g -d
등등 여러가지를 해보지만 뭐 딱히 나오느게 없다. hexo init 를 해보니 이것도 오류다. not empty,...........
그 뒤, s, server, new 이런 것들이 모두 hexo하나만 넣었을 때와 다를바 없이 나온다. 그러다가 테마를 바꾸는걸 따라하려고 git clone 명령어를 치자 주르륵 복사가 된다. 그 뒤론 hexo 명령어가 먹는다. 이런. 뭐지? 싶다.

2. 백업을 따라하다 길을 잃다.

https://futurecreator.github.io/categories/Hexo/ 사이트의 글을 이용하여 백업을 해보려고 하고 있다. 하지만 잘 되지 않는다.

분명 따라서 git commit -a -m 'theme backup' 을 입력했건만 untracked files:…라고 뜬다.
이건 뭔가.. 그래서 구글해본다. untracked git으로..

git기초 수정하고 저장소에 저장하기

근데 여기서는 -a 옵션을 넣으면 되는것 같은데 왜 아직도 untracked…가 뜨는 건지 잘 모르겠다.

여기서 길을 잃는다. 뭔가. 이후 진행이 안된다. 그냥 넘어 가볼까?

이후에도 계속 untracked files: 목록이 나오지만 그냥 가본다. 올라가질 않는다. gitlab에. 초기에 보니 뭐라뭐라 나온다.
git add .을 입력해 본다. 그리고 커밋을 해본다. 뭔가 된다. 올라갔다. 그런데 문제는 다음 것이 안된다.
올라간 파일을 확인해 보니 수정했던 테마폴더가 올라가지 않았다. 이게 뭔가.. 열리지도 않는다. 로컬에서 보니 뭐라고 나온다.

막 이런다. 수정했던 것. 뭐라는건지 모르겠다.

untracked <-> unmodified <-> modified <-> staged <-> commited

출처: https://mobicon.tistory.com/129 [Intelligence Convergence]

이런게 있다. modified 파일이란건데 이걸 staged로 바꿀려면

인것 같은데 왜 안될까?

그뒤로 계속 해도 되지 않는다. 아마 그 테마폴더때문인것 같다.

모르겠다. 초기화 하려고 해도 잘 안되는 것 같고.. 음.. 그렇다.

새로 만들고. 다시 해봐도 hello world는 뜨는데 추가하 글들이 보이지 않는다. 뭔가 문제가 있는것 같다. 암튼 그렇다. 일단 여기까지. 7시 되기 전이 자야겠다. ^^;;
